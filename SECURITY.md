# Política de Seguridad

## Reporte de vulnerabilidades
- Por favor, crea un issue con la etiqueta `security` o contacta a los mantenedores.
- Evita publicar detalles explotables en público antes de un parche.

## Alcance
- Código de la API (FastAPI) y UI (PyQt6).
- Scripts de ejecución y workflows de GitHub Actions.

## Buenas prácticas esperadas
- No exponer credenciales en código o logs.
- Validar entradas de usuario en endpoints.
- Limitar el acceso a archivos de artefactos; revisar permisos.

## Calendario de parches
- Se intentarán publicar parches rápidamente; versiones se agruparán en releases.

## Criptografía y dependencias
- Se acepta disclosure responsable sobre dependencias vulnerables; Dependabot está habilitado.
