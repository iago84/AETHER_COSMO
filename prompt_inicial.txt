AETHERLAB
Subtítulo: Plataforma de simulación, análisis y exploración del campo de memoria del Aether

CONTEXTO GENERAL
El sistema debe materializar una hipótesis de investigación llamada “La memoria del Aether”, entendida como un campo informacional fenomenológico que podría registrar huellas físicas, energéticas y estructurales del universo.

IMPORTANTE:
- No presentar la hipótesis como ciencia confirmada.
- Distinguir siempre entre:
  1. física establecida,
  2. modelo fenomenológico propuesto,
  3. especulación exploratoria.
- El sistema debe ser serio, modular, extensible y útil para investigación computacional real.

OBJETIVO DEL SISTEMA
Construir una plataforma científica que permita:
1. Definir y ejecutar simulaciones del campo Aether.
2. Modelar eventos físicos que “escriben” memoria en el campo.
3. Visualizar la evolución del campo en 2D y 3D.
4. Ingerir datasets reales (Planck, LIGO/GWOSC, SDSS y otros).
5. Aplicar IA para detectar anomalías, patrones residuales y correlaciones.
6. Gestionar experimentos, parámetros, versiones y resultados.
7. Exportar informes técnicos, figuras y tablas.
8. Servir como base futura para papers, tesis, notebooks y paneles de análisis.

Modelo base a implementar:
Campo A(x,t) o A(x,y,z,t)
Ecuación inicial de trabajo:
□A + λA = α(TμνTμν) + βR² + γQ
y también una versión simplificada computacional:
∂A/∂t = S(x,t) - λA + D∇²A + N(x,t)

Quiero:
- implementación de ambas capas:
  1. modelo teórico simbólico/documentado
  2. modelo numérico simplificado ejecutable
- documentación interna de qué parte es formal y qué parte es aproximación
- varios tipos de fuentes S(x,t):
  - supernova
  - fusión de agujeros negros
  - pulso puntual
  - ruido estocástico
  - fuente periódica
  - fuente importada desde dataset
- varios kernels espaciales:
  - gaussiano
  - exponencial
  - lorentziano
  - top-hat
- varios regímenes de disipación:
  - memoria corta
  - media
  - larga
  - casi permanente
